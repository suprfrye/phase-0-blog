What is Functional Programming?

Sunday, February 15, 2015

Functional programs use functions as their building blocks, while object-oriented programs utilize objects. Functional programming is:

  1. Immutable: the data should not change. Opts to create new data rather than modify the original.
  2. Stateless: not aware of previous executions.

Functional programs avoid mutable data and changing state. They focus on eliminating side effects and creating simple, abstract functions which makes the program easier to understand, and more importantly, easier to predict the behavior of. This way calling a function with the same argument will produce the same output every time.

Functional programming utilizes referential transparency, the substitution of an expression with its value without changing how the program works. This helps to verify and ensure the results as well as optimize the program.
